In this section we introduce step by step the installation procedure of CasADi. First we detail what software packages need to be installed in order to use CasADi.

Software requirements

CasADi consists of a symbolic core, which is strongly used by the interfaces that are shipped with it. Now we summarize what the dependencies of each are. As a general rule you will always need a C++ compiler (tested with gcc 4.4.3) and CMake (tested with version 2.8.2).

Package/interface Dependency
CasADi core None
CSparse interface CSparse (comes with CasADi)
Ipopt interface Ipopt
KNITRO interface KNITRO
LAPACK interface library with LAPACK api
LiftOpt interface LiftOpt
Sundials interface SUNDIALS
SuperLU interface SuperLU, library with BLAS api
CasADi for python SWIG

Compilation

First, you have to get CasADi from http://www.casadi.org/ or check out the git repository from GitHub.

$ git clone https://github.com/casadi/casadi.git -b tested casadi

Set the environment variable CMAKE_PREFIX_PATH to inform CMake where the dependecies are located. For example if Sundials headers and libraries are installed under $HOME/local/, then type

export CMAKE_PREFIX_PATH=$HOME/local/

Go to directory of the source tree, create a directory called e.g. ./build where all compilation-related files will be created.

cd casadi; mkdir build; cd build

Generate the Makefile by typing

cmake ..

Check the output of CMake if the dependencies are correctly located. The interfaces without underlying libraries and headers won't be compiled. Adjust CMake variables (e.g. CMAKE_INSTALL_PREFIX if you want to install headers and libraries) by typing

ccmake ..

Now compile the C++ libraries.

make

And install headers and libraries

make install

If you wish to use the python interface you can compile it with the python target.

make python
